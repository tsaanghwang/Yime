# 项目开发路线图（ROADMAP）

## 愿景
打造高效、易用、可扩展的音元输入法，支持多平台和多种输入模式，推动汉语输入体验创新。

---

## 近期目标（2026 Q2）

### 1. 补全和扩展单元测试
- 覆盖核心 Python 代码（韵母/声母/音节转换、数据库操作、数据导入导出等）
- 增加前端（React/TypeScript）组件的自动化测试
- 持续修复测试中发现的 bug

### 2. 完善和补充项目文档
- 编写/补全 API 说明、数据库结构说明、数据流文档
- 增加开发者快速上手指南和贡献流程说明
- 完善 FAQ 和常见问题解答

### 3. 优化韵母转换与输入逻辑
- 优化韵母、声母、音节的转换算法，提升准确性和健壮性
- 检查并完善多输入模式（全拼、简拼、双拼、并击）切换逻辑
- 优化候选词排序、用户词库管理等功能

### 4. 前端 UI/UX 优化
- 提升 Web 端界面美观性和交互体验
- 优化响应速度和移动端适配
- 增加输入法使用引导和可视化反馈

### 5. 配置持续集成与自动测试
- 配置 GitHub Actions 或其他 CI 工具，实现自动化测试、构建和发布
- 增加代码风格检查（如 ESLint、黑格式化等）

### 6. 收集用户反馈并完善 FAQ
- 建立用户反馈渠道（如 Issues、Discussions）
- 定期整理和归纳用户反馈，完善 FAQ 文档

### 7. 制定和发布开发路线图
- 动态维护本 ROADMAP，根据实际进展和反馈调整优先级

---

## 中期目标（2026 Q3-Q4）
- 前端 UI/UX 升级，支持主题切换和自定义皮肤
- 增加用户自定义词库和导入导出功能
- 跨平台支持（移动端/桌面端原型）
- 优化数据库结构，提升性能和可扩展性
- 增加输入法配置界面和个性化设置

---

## 长期目标
- 智能联想与语义输入
- 国际化与多语言支持
- 语音输入与语音识别集成
- 开放 API，支持第三方插件和扩展
- 建立完善的社区生态和贡献体系

---

> 本路线图将根据项目进展和用户反馈持续更新。
