Yime 帮助

这份帮助页面向普通用户，集中说明 Yime 在 PIME 中的常用入口、候选操作、反查显示和独立工具方向。帮助、设置、诊断这些较重的界面都尽量放在独立窗口里，语言栏只保留轻量分发。

语言栏轻量化 独立工具优先 避免宿主不稳定

快速开始

  • 在 PIME 语言栏中切换到 Yime 方案。
  • 输入编码后，用空格、回车或候选序号上屏。
  • 方向键、Page Up、Page Down、Home、End 可用于移动候选。
  • 按 Esc 取消当前组字。

菜单说明

  • 打开工具箱：进入独立工具入口，适合承载设置、诊断、词库管理等较复杂操作。
  • 设置：可以切换变长、等长、省键方案，以及中英模式、全半角、标点、候选排列和候选项数。
  • 重新部署 Rime：让已经写到磁盘的配置重新编译生效。它不是“重新导入系统词库”。
  • 同步 Rime 用户数据：调用 Rime 自带的用户数据同步，只处理 Rime 管理的数据，不包含 Yime 独有的界面偏好状态。
  • 显示编码：控制候选上的反查注释显示方式。
  • 用户词库：打开独立词库管理器，适合做增删改查、导入导出、冲突预览和应用词库。
  • 帮助：查看主帮助、试用反馈说明，或复制试用反馈模板。

反查显示

  • 隐藏编码：只保留候选本身。
  • 标准拼音：显示带声调的标准拼音。
  • 音元拼音：显示当前 Yime 方案对应的拼音编码。
  • 键位序列：显示实际键位序列。

数字声调拼音主要用于录入用户词条，例如 lv4、ri4 ben3，不作为日常候选显示模式。

独立工具方向

  • 词库管理已经是独立窗口，不再强行塞进语言栏回调路径。
  • 设置、诊断、日志和帮助也优先走同样的模式。
  • 这样做的目标是减少焦点丢失、菜单回调过重和宿主程序异常退出的风险。
  • 如果某个功能需要更完整的交互，优先考虑扩展工具箱，而不是继续加重 TSF 菜单逻辑。

常用路径

  • 用户 Rime 数据：%APPDATA%\PIME\Rime
  • PIME 日志：%LOCALAPPDATA%\PIME\Logs
  • Yime 共享运行时数据：go-backend\input_methods\yime\data
