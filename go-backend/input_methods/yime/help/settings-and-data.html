设置与数据

这页帮助主要解释 Yime 设置项写到哪里、哪些操作只是保存文件、哪些操作会触发重新构建，以及为什么独立工具窗口不保证继续保持当前输入法激活。

主要目录

  • 用户配置与用户词库：%APPDATA%\PIME\Rime
  • 日志目录：%LOCALAPPDATA%\PIME\Logs
  • 共享运行时数据：go-backend\input_methods\yime\data

语言栏里的维护项

  • 重新部署 Rime：重新编译并部署当前运行时配置，让已经写入的方案文件或配置生效。
  • 同步 Rime 用户数据：调用 Rime 自带的同步能力，适用于 Rime 自己管理的数据。
  • 打开数据与日志目录：直接打开用户目录、共享目录、同步目录和日志目录，方便检查“文件有没有真的写进去”。

注意：yime_settings_state.json 这类 Yime 独立界面状态不属于 Rime 原生同步范围。

为什么有些设置需要“应用并重建”

像方案切换、候选项数这类设置，不只是改界面显示，还需要让 Rime 运行时重新读取并编译相关配置。所以单纯保存文件可能不够，通常需要再执行一次重建或重新部署。

  • 保存文件：把设置写到用户目录中的配置文件。
  • 应用并重建：保存后再调用 rime_deployer.exe，让运行时真正看到新值。

独立窗口与输入法切换

Yime 的设置页、词库管理器、诊断页都是外部窗口。Windows 在切换到另一个顶层窗口时，可能恢复那个窗口自己的输入法上下文，而不是延续刚才宿主应用里的 Yime。

这不是产品“忘记了保持输入法”，而是刻意避免用更激进的 TSF 或焦点操作去强拉输入法状态。那样虽然看起来“更自动”，但更容易带来焦点跳动、菜单异常或宿主不稳定。
