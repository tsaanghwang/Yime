诊断说明

Yime 诊断工具的目标不是只展示路径，而是帮你判断“命令有没有打到后端、部署有没有真正发生、当前运行的是不是预期安装、问题更像配置没写进去还是运行时没刷新”。

优先检查的几层

  • %LOCALAPPDATA%\PIME\Logs：看后端日志有没有请求、命令、部署和错误信号。
  • %APPDATA%\PIME\Rime：看用户配置和用户词库有没有真的写到磁盘。
  • 已安装的 server.exe 与 rime_deployer.exe：确认问题不是“源码修了但安装没更新”。
  • 正在运行的 PIMELauncher 与 server.exe：确认进程已经重启到新版本。

诊断工具当前会做什么

  • 检查用户目录、共享目录、帮助目录、日志目录是否存在。
  • 检查安装路径、运行中的 PIMELauncher / server.exe、以及关键二进制修改时间。
  • 检查 default.custom.yaml、custom_phrase.txt、yime_user_phrases.txt、yime_settings_state.json。
  • 解释最近的 commandId=、部署信号、错误样式日志和它们之间的时间先后关系。
  • 生成可复制的结构化报告，支持匿名化、预设和日志摘录。

看到“设置无效”时先怎么判断

  • 先看日志里有没有对应的 onCommand 或 commandId。
  • 再看用户目录下的文件有没有被写到正确的值，例如 menu/page_size。
  • 最后看部署或重建信号有没有跟在命令之后出现。

如果命令到了、文件也改了，但部署没发生，问题往往不在菜单点击本身，而在部署链路或运行时刷新。

本地调试辅助

仓库侧还有一个开发者辅助脚本：go-backend\run_admin_yime_tests.cmd。它用于本地调试和提权测试，不属于已安装给普通用户使用的运行时功能。
