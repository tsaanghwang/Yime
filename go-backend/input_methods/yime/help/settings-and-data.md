# 设置与数据

## 输入方案

音元输入法提供三种编码模式：

| 方案 | 名称 | 编码长度 | 特点 |
|------|------|----------|------|
| 变长 | yime_variable | 2-4 键 | 默认方案，编码长度随音节变化 |
| 等长 | yime_full | 固定 4 键 | 每音节等长，无重码 |
| 省键 | yime_shorthand | 1-3 键 | 最短编码，需安装省键方案数据 |

通过语言栏「模式」菜单或「设置工具」切换方案。Windows 输入法列表中的显示名为 **音元**。

## 候选设置

### 候选项数

支持 5-9 个候选项，默认 5 个。通过语言栏「候选项数」菜单或设置工具调整。

修改候选项数会触发 Rime 会话重建，当前输入会被清除。这是设计行为——修改后请重新输入。

### 候选排列

支持竖排（默认）和横排。通过语言栏 **「横竖切换」** 按钮或设置工具切换。

### 语言栏切换按钮

三个切换按钮使用固定标签，状态由图标表示：

| 按钮 | 标签 | 图标含义 |
|------|------|----------|
| 中西切换 | 中西切换 | 中文 / 西文输入模式 |
| 全半切换 | 全半切换 | 半角 / 全角字符 |
| 横竖切换 | 横竖切换 | 竖排 / 横排候选 |

### 候选选择键

| 按键 | 选择 |
|------|------|
| Space | 第 1 个候选 |
| `` ` `` | 第 2 个候选 |
| `-` | 第 3 个候选 |
| `=` | 第 4 个候选 |
| `\` | 第 5 个候选 |

当候选项数超过 5 个时，第 6-9 个候选项需用鼠标点击选择。

### 反查显示

控制候选窗中汉字编码注释的显示方式：

| 模式 | 说明 |
|------|------|
| 键位序列 | 显示 Rime 返回的原始编码注释（默认） |
| 隐藏编码 | 不显示编码注释 |
| 标准拼音 | 显示带调号的标准拼音（如 zhōng） |
| 音元拼音 | 显示音元编码 |

## 数据目录

### 用户数据

`%APPDATA%\PIME\Rime\`

| 文件 | 说明 |
|------|------|
| `default.custom.yaml` | 方案选择和候选项数覆盖 |
| `yime_variable.custom.yaml` | 变长方案自定义配置 |
| `yime_full.custom.yaml` | 等长方案自定义配置 |
| `yime_shorthand.custom.yaml` | 省键方案自定义配置 |
| `user.yaml` | Rime 用户状态（上次选择的方案） |
| `yime_user_phrases.txt` | 用户词库源文件 |
| `custom_phrase_variable.txt` | 变长模式 Rime 用户词库（自动生成） |
| `custom_phrase_full.txt` | 等长模式 Rime 用户词库（自动生成） |
| `custom_phrase_shorthand.txt` | 省键模式 Rime 用户词库（自动生成） |
| `yime_settings_state.json` | 独立 UI 偏好（反查模式、候选排列） |
| `yime_user_blocklist.txt` | 用户屏蔽词表源文件 |

### 共享数据

安装目录下 `go-backend\input_methods\yime\data\`

包含方案定义、词典、编码映射表、拼音归一化表等运行时数据。

### 日志

`%LOCALAPPDATA%\PIME\Logs\`

- `go_backend.log` — Go 后端主日志

## 语言栏维护命令

- **重新部署 Rime**：重新运行 Rime 运行时部署，使修改后的方案文件或配置生效。这不是「重新导入系统词库」。
- **同步 Rime 用户数据**：调用 Rime 原生用户数据同步功能。仅同步 Rime 管理的数据，不包括 `yime_settings_state.json` 等 Yime 独立状态。
- **打开数据与日志文件夹**：直接打开用户数据、共享数据、同步和日志目录。

## 用户词库

### 添加词条

1. 通过语言栏「用户词库」按钮或工具箱「词库管理」打开词库管理器
2. 输入汉字和数字标调拼音（如 `zhong1 guo2`）
3. 系统自动将拼音转换为音元编码
4. 写入 `yime_user_phrases.txt` 并生成对应模式的 `custom_phrase_*.txt`

### 词库格式

源文件 `yime_user_phrases.txt`：

```
词条<TAB>数字标调拼音<TAB>权重
中国	zhong1 guo2	1000000
```

生成文件（以变长模式为例）`custom_phrase_variable.txt`：

```
词条<TAB>音元编码<TAB>权重
中国	7dgo	1000000
```

### 注意事项

- 应用词库时会为三种方案各生成独立的 `custom_phrase_{mode}.txt`。
- 拼音中 `ü` 可用 `v` 或 `u:` 代替。
- 默认权重为 1000000，用户词会优先于主词典显示。

## 用户屏蔽词表

被加入屏蔽词表的词条不会出现在输入候选中。

1. 通过工具箱「用户屏蔽词表」打开 `blocklist-manager.exe`
2. 编辑 `yime_user_blocklist.txt`
3. 保存后立即影响后续候选过滤

## 工具箱

语言栏「工具」按钮打开 `tool-hub.exe`，当前包含：

| 入口 | 说明 |
|------|------|
| 词库管理 | 用户词库增删改查与导入 |
| 反查编码 | 汉字反查拼音与音元编码 |
| 系统词库审查 | 只读扫描已安装系统词库 |
| 用户屏蔽词表 | 管理个人屏蔽词 |
| 设置工具 | 方案、候选项数、反查、排列等 |
| 诊断工具 | 路径/进程/日志检查与报告导出 |
| 用户数据目录 / 共享数据目录 / 帮助 | 打开对应目录或帮助文档 |

## 设置工具

独立 Win32 窗口（`settings-tool.exe`），提供：

- 输入方案选择
- 候选项数调整
- 反查显示模式
- 候选排列方式
- 「应用并重建」按钮

修改方案或候选项数后，需点击「应用并重建」使配置生效。该操作会：

1. 写入 `default.custom.yaml`、`user.yaml`、`yime_settings_state.json`
2. 调用 `rime_deployer.exe` 重建 Rime 运行时数据
