# 设置与数据

## 输入方案

音元输入法提供三种编码模式：

| 方案 | 名称 | 编码长度 | 特点 |
|------|------|----------|------|
| 变长 | yime_variable | 2-4 键 | 默认方案，编码长度随音节变化 |
| 等长 | yime_full | 固定 4 键 | 每音节等长，无重码 |
| 省键 | yime_shorthand | 1-3 键 | 最短编码，需安装省键方案数据 |

三种方案都启用连续输入和 Rime 整句组句。变长、省键派生码保留虚首音，作为自动分词边界。虚首音
对应零声母，包括拼音书写中的隔音符号 `'` 以及 `y`、`w`；实首音则对应非零声母。

通过语言栏「模式」菜单或「设置工具」切换方案。Windows 输入法列表中的显示名为 **音元**。

## 候选设置

### 候选项数

支持 5-9 个候选项，默认 5 个。通过语言栏「候选项数」菜单或设置工具调整。

修改候选项数会触发 Rime 会话重建，当前输入会被清除。这是设计行为——修改后请重新输入。

### 候选排列

支持竖排（默认）和横排。通过语言栏 **「横竖」** 按钮或设置工具切换。

### 语言栏切换按钮

三个切换按钮使用固定标签，状态由图标表示：

| 按钮 | 标签 | 图标含义 |
|------|------|----------|
| 中西文切换 | 中西 | 中文 / 西文输入模式 |
| 全半角切换 | 全半 | 半角 / 全角字符 |
| 横竖排切换 | 横竖 | 竖排 / 横排候选 |

### 工具入口

- 使用桌面语言栏时，可直接点击「用户词库」「反查编码」和「工具中心」。
- 语言栏停靠在任务栏时，独立按钮可能不会显示。左键单击「中」按钮，在弹出的「设置」菜单中直接选择「用户词库」「反查编码」或「工具中心」。
- 两处入口使用相同命令：「工具中心」打开的窗口名称仍为「Yime 工具箱」。

### 候选选择键

| 按键 | 选择 |
|------|------|
| Space / Enter / Shift+1 | 第 1 个候选 |
| Shift+2 | 第 2 个候选 |
| … | … |
| Shift+9 | 第 9 个候选 |

候选窗显示 `⇧1`…`⇧9`，箭头表示必须按住 Shift。与流行拼音输入法不同，Yime 有意不采用裸数字键选词；Base 层 `0`—`9` 十个数字键始终用于输入编码，候选出现时也不切换含义，Shift+0 不选词。物理键盘上的对应键面是 `! @ # $ % ^ & * (`，布局图会照实标出这些标点，但它们不直接充当候选窗序号，因为连续标点不易辨认为顺序标签。

Base 层的 `-`、`=`、`\` 是新布局编码键，不再选择候选。

### 反查显示

控制候选窗中汉字编码注释的显示方式：

| 模式 | 说明 |
|------|------|
| 键位序列 | 显示 Rime 返回的原始编码注释（默认） |
| 隐藏编码 | 不显示编码注释 |
| 标准拼音 | 显示带调号的标准拼音（如 zhōng） |
| 音元拼音 | 用 PUA 音元字符显示候选的音元拼音；不改变实际按键编码 |

## 数据目录

### 用户数据

`%APPDATA%\PIME\Rime\`

| 文件 | 说明 |
|------|------|
| `default.custom.yaml` | 方案选择和候选项数覆盖 |
| `yime_variable.custom.yaml` | 变长方案自定义配置 |
| `yime_full.custom.yaml` | 等长方案自定义配置 |
| `yime_shorthand.custom.yaml` | 省键方案自定义配置 |
| `user.yaml` | Rime 用户状态（上次选择的方案） |
| `yime_user_phrases.txt` | 用户词库源文件 |
| `custom_phrase_variable.txt` | 变长模式 Rime 用户词库（自动生成） |
| `custom_phrase_full.txt` | 等长模式 Rime 用户词库（自动生成） |
| `custom_phrase_shorthand.txt` | 省键模式 Rime 用户词库（自动生成） |
| `yime_settings_state.json` | 独立 UI 偏好（反查模式、候选排列） |
| `yime_blocklist.txt` | 用户屏蔽词表源文件 |

### 共享数据

安装目录下 `go-backend\input_methods\yime\data\`

包含方案定义、词典、编码映射表、拼音归一化表等运行时数据。

### 日志

`%LOCALAPPDATA%\PIME\Logs\`

- `go_backend.log` — Go 后端主日志

## 语言栏维护命令

- **数据维护**：集中放置「同步数据…」「重新部署…」和「打开目录」；两项执行型维护命令都会再次确认。
- **重新部署…**：仅在手工修改或恢复配置、排查旧编译缓存时使用。确认后由外部部署器在后台构建并校验当前方案，完成后只在安全请求边界重建当前会话。这不是「重新导入系统词库」，不会恢复默认词库或删除用户词库；重复点击不会启动第二个任务。
- **同步数据…**：确认后调用 Rime 原生用户数据同步功能。仅同步 Rime 管理的学习数据，不包括 `yime_settings_state.json`、YIME 用户词库源文件或屏蔽词表。
- **打开目录**：打开用户数据、共享数据、同步和日志目录。

## 用户词库

### 添加词条

1. 通过桌面语言栏「用户词库」按钮、任务栏「设置 → 用户词库」或工具箱「词库管理」打开词库管理器
2. 输入汉字和数字标调拼音（如 `zhong1 guo2`）
3. 系统自动将拼音转换为音元编码
4. 点击「保存」后可继续添加下一条；点击「退出」、窗口 X 或取消才结束添加
5. 写入 `yime_user_phrases.txt`，并生成三种模式各自的 `custom_phrase_*.txt`

系统初始词库中已有的词条会收到警示并被拒绝添加。校验失败时，当前输入会保留在对话框中供修改。

添加和编辑对话框内可直接调整权重：在「减」和「加」之间输入每次调整的数值，每点击一次按该数值加减一次。

添加、编辑、删除、撤销、应用、导入和错误提示均使用中文操作按钮；确认按钮组居中，不再显示系统 `OK` 或 `Yes/No`。

### 词库格式

源文件 `yime_user_phrases.txt`：

```
词条<TAB>数字标调拼音<TAB>权重
中国	zhong1 guo2	1000000
```

生成文件（以变长模式为例）`custom_phrase_variable.txt`：

```
词条<TAB>音元编码<TAB>权重
中国	7dgo	1000000
```

### 注意事项

- 应用词库时会为三种方案各生成独立的 `custom_phrase_{mode}.txt`。
- 拼音中 `ü` 可用 `v` 或 `u:` 代替。
- 默认权重为 1000000，用户词会优先于主词典显示。

## 用户屏蔽词表

被加入屏蔽词表的词条不会出现在输入候选中。

1. 通过工具箱「用户屏蔽词表」打开 `blocklist-manager.exe`
2. 可添加、搜索、多选删除、导入、导出，并撤销最近一次改动
3. 所有成功改动会立即写入 `yime_blocklist.txt`，影响后续候选过滤

## 工具箱

桌面语言栏「工具中心」按钮或任务栏「设置 → 工具中心」会打开 `tool-hub.exe`；其窗口名称为「Yime 工具箱」，当前包含：

| 入口 | 说明 |
|------|------|
| 高级布局 | 复制、试打、保存并安全应用个人音元键盘布局 |
| 词库管理 | 用户词库增删改查与导入 |
| 反查编码 | 汉字反查拼音与音元编码 |
| 系统词库审查 | 只读扫描已安装系统词库 |
| 用户屏蔽词表 | 管理个人屏蔽词 |
| 设置工具 | 方案、候选项数、反查、排列及用户数据备份恢复 |
| 诊断工具 | 路径、进程、日志检查与报告导出 |
| 用户数据目录 / 共享数据目录 | 打开用户数据或已安装共享数据 |
| 查看帮助 / 反馈说明 | 打开本地帮助和问题反馈指南 |

“高级布局”默认使用官方布局作为只读基础，个人方案保存在 `%APPDATA%\PIME\Rime\yime_layouts`。确认应用时，等长、变长、省键三种模式都会同步重建系统词典和用户自建词表，并迁移各自的学习库；三套方案全部编译成功后才通知正在运行的输入会话统一刷新。失败时恢复原覆盖文件，官方安装数据不会被修改。

## 设置工具

独立 Win32 窗口（`settings-tool.exe`），提供：

- 输入方案选择
- 候选项数调整
- 反查显示模式
- 候选排列方式
- 「应用」按钮
- 可移植用户数据「备份」和「恢复」按钮

修改方案或候选项数后，需点击「应用」使配置生效。该操作会：

1. 写入 `default.custom.yaml`、`user.yaml`、`yime_settings_state.json`
2. 调用 `rime_deployer.exe` 重建 Rime 运行时数据

「备份」会在 Windows“文档\YIME 备份”中创建带时间戳、文件清单和 SHA-256 校验的可移植用户数据快照。它包含 YIME 设置、用户词库、屏蔽词表和 Rime `sync` 同步数据，但不包含可重新生成的 `build`、临时通知文件，以及输入法运行期间被锁定且不能一致复制的 `*.userdb` LevelDB 目录。

「恢复」使用最近一次手工备份。恢复前会自动创建当前数据的安全快照；校验并恢复文件后，工具会重新构建 Rime，并通知当前 YIME 会话重新读取设置和词库。
