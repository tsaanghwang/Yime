# Yime 故障排除指南

本文档面向普通用户，解决 Yime 输入法常见问题。

## 输入法不出现

**症状**：切换到 Yime 后，输入字母没有候选窗口。

**排查步骤**：

1. **等待初始化**：首次切换后需 10-30 秒加载 Rime 引擎，请耐心等待
2. **检查进程**：打开任务管理器，查看是否有 `server.exe` 进程运行
   - 如果没有：PIMELauncher 可能未启动，尝试重启电脑
   - 如果有但 CPU 占用 0%：server.exe 可能已崩溃，见下方"重启后端"
3. **检查语言栏**：右键语言栏图标，确认 Yime 已被选中
4. **重启后端**：打开任务管理器，结束 `server.exe` 进程，然后重新切换到 Yime

## 语言栏菜单消失

**症状**：右键语言栏没有 Yime 的菜单项。

**排查步骤**：

1. 确认 `server.exe` 正在运行（任务管理器）
2. 结束 `server.exe`，重新切换到 Yime
3. 如果重装后出现此问题，确认安装路径正确（`C:\Program Files (x86)\YIME\`）
4. Windows Smart App Control 可能阻止了 server.exe，见下方"Smart App Control 问题"

## 候选窗口不显示

**症状**：输入编码后没有候选列表。

**排查步骤**：

1. 确认当前应用是文本输入场景（记事本、浏览器搜索框等）
2. 尝试按 `Esc` 取消当前输入，重新输入
3. 检查候选排列设置是否被意外改为隐藏模式

## 设置不生效

**症状**：修改候选数或排列方式后没有变化。

**排查步骤**：

1. 修改后等待 2-3 秒，Rime 引擎需要重新部署
2. 如果仍未生效，尝试切换到其他输入法再切回 Yime
3. 检查 `%APPDATA%\PIME\Rime\` 下对应的 `.custom.yaml` 文件是否已更新

## 用户词库不生效

**症状**：添加的用户词没有出现在候选中。

**排查步骤**：

1. 在词库管理器中添加词条后，必须点击"应用"按钮
2. 应用后等待 2-3 秒让 `rime_deployer.exe` 完成重新部署
3. 用户词库会同时生成并部署到三种方案，无需按方案重复添加
4. 若当前已激活的输入会话仍显示旧结果，重新激活输入法或点击语言栏「重新部署 Rime」刷新会话

## 反查工具启动慢

**症状**：打开反查编码工具后窗口显示"正在加载"较长时间。

**说明**：反查工具需要加载约 46 万条系统词库，首次加载需要几秒钟。加载过程中会显示进度条和状态提示，请耐心等待。加载完成后后续查询即时响应。

## Smart App Control 问题

**症状**：安装程序启动后中途退出、重装后语言栏菜单消失、`server.exe` 无法启动，或工具箱提示“An Application Control policy has blocked this file”。

**原因**：Smart App Control 会阻止信誉未知且未由受信任提供商签名的可执行文件。安装器即使能够启动，内部调用的部署工具、配置工具、DLL 注册程序或卸载程序仍可能被单独阻止，表现为安装中止。VERSIONINFO 有助于识别文件，但不能替代可信代码签名。若构建把每次提交号写入 EXE，无关改动也会产生新哈希并丢失既有信誉。

**解决**：
- 确保使用最新版本的 Yime 安装包
- 检查被阻止 EXE 的“属性 → 详细信息”和“数字签名”；正式发布物应同时有稳定版本信息和可信签名
- 在“事件查看器 → Microsoft → Windows → CodeIntegrity → Operational”中确认事件 3033/3077/3118 指向的具体文件
- 区分开发包与发行包：`YIME-unsigned-test-installer` 只供开发测试，不能据此判断正式签名链是否有效
- 如果没有 CodeIntegrity 阻止事件，再检查安装日志、管理员提权、文件锁和杀毒软件隔离，不要把所有安装退出都归因于证书
- 不建议为绕过单个工具而关闭 Smart App Control；开发构建应保持可复现，发布构建应配置 Trusted Signing 或受信任 CA 证书

## 重启后端

当 Yime 行为异常时，重启后端通常能解决问题：

1. 打开任务管理器（Ctrl+Shift+Esc）
2. 在"详细信息"标签页找到 `server.exe`
3. 右键 → 结束任务
4. 在任意文本框中切换到 Yime，server.exe 会自动重新启动

如果 server.exe 无法自动重启：

1. 在任务管理器中找到并结束 `PIMELauncher.exe`
2. 手动运行 `C:\Program Files (x86)\YIME\PIMELauncher.exe`
3. 切换到 Yime

## 收集诊断信息

如果以上方法无法解决问题，请收集以下信息用于反馈：

1. **日志文件**：`%APPDATA%\PIME\Rime\go_backend.log`
2. **系统信息**：Windows 版本（设置 → 系统 → 关于）
3. **Yime 版本**：语言栏帮助菜单中查看
4. **复现步骤**：详细描述从正常到异常的操作过程
5. **截图**：异常状态的截图

可通过语言栏的"工具箱 → 诊断工具"自动收集部分诊断信息。

## 常见错误信息

| 错误信息 | 原因 | 解决方法 |
|----------|------|----------|
| "找不到拼音编码表" | 共享数据目录缺失 | 重新安装 Yime |
| "Rime 初始化失败" | 用户数据目录损坏 | 备份后删除 `%APPDATA%\PIME\Rime\build\`，重启 Yime |
| "应用用户词库失败" | 词库格式错误 | 用词库管理器检查并修复词条格式 |
