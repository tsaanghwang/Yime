# Yime 原生 Win32 UI 规范

本文档适用于 `go-backend/cmd/` 下的设置、诊断、反查、词库、系统词库审查、屏蔽词表和工具箱。目标是让工具保持一致、可扫描、无闪烁，并避免 UI 逻辑重新进入 TSF 回调或 PowerShell 运行时路径。

## 1. 架构规则

- 普通用户的复杂 UI 使用独立 Go + Win32 EXE
- 语言栏只做轻量命令分发，不在 TSF 回调中创建复杂窗口
- 运行时不得启动或嵌入 PowerShell；PowerShell 仅用于开发、测试、构建、安装和维护
- 同一工具只能存在一个主窗口；再次启动应激活已有窗口
- 主窗口显示使用 `win32ui.PresentMainWindowAfterLaunch`，避免启动闪烁

## 2. 内容布局

推荐基础尺寸：

- 外边距：12-16 px
- 同排控件间距：8-12 px
- 输入和按钮高度：26-30 px
- 标签与输入框垂直间距：4-8 px

规则：

1. 用内容边界计算客户区宽高，不维护与控件无关的独立窗体尺寸
2. 同一内容区的各横排使用相同左、右边界
3. 固定功能控件使用稳定宽度，剩余空间分配给搜索框、列表和详情区
4. 下拉框创建高度可包含展开区，但布局测试按可见行高判断
5. 固定格式组件使用明确尺寸，动态文字不能推动其它控件换位

反查工具的标准顶部顺序：

```text
查询词条 | 输入框 | 包含匹配 | 方案 | 下拉框 | 查询
```

## 3. 标签与文字

- 标签必须按实际文字宽度创建，避免 STATIC 控件在文字后留下灰色长条
- 使用 `GetTextExtentPoint32W` 或等价 Win32 测量，不用字符数硬编码中文宽度
- 为测量结果保留约 4 px 水平余量
- 长说明文字允许换行；按钮文字不得截断
- 中文界面不混用 `OK`、`Yes/No`、`Cancel` 等英文系统标签

推荐操作词：

| 场景 | 标签 |
|------|------|
| 保存并留在连续流程 | 保存 |
| 接受一次性操作 | 确认 |
| 放弃当前操作 | 取消 |
| 结束连续添加 | 退出 |
| 删除前确认 | 确认 / 取消 |
| 导入方式 | 完全替换 / 合并导入 / 取消 |

## 4. 按钮布局

- 对话框底部按钮作为一个组在内容区水平居中
- 同组按钮间距保持一致
- 两字按钮建议宽 78-88 px；较长文字根据测量结果增加内边距
- “确认”在左、“取消/退出”在右，除非 Windows 平台既有行为明确要求不同顺序
- 单按钮通知框使用居中的“确认”
- 图标按钮优先使用项目现有图标库；纯 Win32 工具没有图标资源时保持文本命令清晰

## 5. 对话框行为

- 所有模态框必须设置 owner，并在关闭后恢复 owner 的前台和启用状态
- 取消按钮和窗口 X 返回同一取消结果，不触发字段必填校验
- 连续添加中，“保存”完成当前词条后打开下一条；“退出”、取消或 X 才结束
- 校验失败保留用户已输入内容，不关闭整个管理器
- 破坏性操作必须有中文确认框
- 信息、警告、错误按标题区分，例如“操作完成”“提示”“操作失败”
- 词库管理器的运行路径不得调用系统 `MessageBoxW` 产生 `OK/Yes/No`

## 6. 焦点与键盘

- 输入框、下拉框和动作按钮使用 `WS_TABSTOP`
- Tab 顺序与视觉顺序一致
- Enter 只触发明确的默认动作；Esc 等同取消
- 模态消息循环使用 `IsDialogMessageW`，保证 Tab、Enter、Esc 正常
- 激活已有窗口时恢复最小化状态并置前，不创建重复进程

## 7. 状态与长任务

- 数据加载、部署和导入不能让窗口表现为“未响应”
- 长任务放到后台 goroutine，完成后通过自定义窗口消息回到 UI 线程
- 加载状态、结果数量、截断和错误应显示在稳定的状态区域
- 状态文字变化不能改变主布局尺寸
- 启动外部部署器必须隐藏控制台窗口

## 8. 测试要求

布局计算应抽成纯函数，至少覆盖：

- 顺序正确、无重叠
- 各横排等宽
- 内容边界与客户区一致
- 按钮组真正居中
- 控件间距一致
- 空值、最长标签和整数边界

测试通过后还要：

1. 构建带 VERSIONINFO 的正式 EXE
2. 更新到安装目录
3. 从工具箱或语言栏真实入口打开
4. 检查 Smart App Control、焦点、Tab、取消和重复启动
5. 比较构建与安装文件 SHA-256

## 9. 审查清单

- [ ] 没有运行时 PowerShell
- [ ] 标签按文字宽度收紧
- [ ] 同排控件左右边界一致
- [ ] 按钮组居中且中文化
- [ ] X、取消、退出语义明确
- [ ] 校验错误不级联关闭主 UI
- [ ] 长任务不阻塞消息循环
- [ ] 有纯布局或行为回归测试
- [ ] 正式构建和安装态均已验证
