Yime 本地编译与重装

在浏览器里打开本页对照操作，不会把说明误输入到编辑器或终端。 仓库路径：C:\dev\Yime · 快捷打开：Open-Dev-Guide.cmd

日常开发（推荐）

在仓库根目录依次执行：

Set-Location C:\dev\Yime
.\Build.cmd
.\Reinstall-PIME-Test.cmd
命令做什么
Build.cmd Build.ps1 → build.bat（Win32 / x64 / 可选 ARM64 C++、go-backend、McBopomofo）→ NSIS 安装包
Reinstall-PIME-Test.cmd 强制版开发重装：自动预检、DLL 被锁时原地升级、成功不暂停；装到 C:\Program Files (x86)\YIME，含 go-backend
装完后：先切到别的输入法，再切回「音元拼音」，语言栏才会刷新。

只改了 Go 后端（更快）

Set-Location C:\dev\Yime\go-backend
cmd /c build.bat
Set-Location C:\dev\Yime
.\Reinstall-PIME-Test.cmd

不必每次全量跑 Build.cmd。

重装脚本：强制版（勿改回简化版）

根目录的 Reinstall-PIME-Test.cmd 是副本，每次 Build.cmd 结束时会从模板覆盖。要改流程请改模板：

正本（唯一应编辑的文件）：
  tools\templates\Reinstall-PIME-Test.cmd

同步到根目录：
  tools\refresh-dev-test-cmds.ps1
  （build.bat 在每次完整编译后自动调用）

根目录副本（会被覆盖，不要当正本改）：
  Reinstall-PIME-Test.cmd

强制版流程（当前行为）

  1. 自动请求管理员权限（UAC）
  2. 调用 dev-stop-pime.ps1 -Auto：不等待「按 Enter」；尽量结束 PIMELauncher / go-backend
  3. 若 PIMETextService.dll 仍被加载（常见：explorer.exe、Cursor、浏览器）→ 退出码 2 → 跳过完整卸载
  4. 否则先跑 dev-uninstall.ps1；若卸载报错也自动继续原地安装（不再问 Y/N）
  5. 跑 dev-install.ps1：复制 DLL / PIMELauncher / python / node / go-backend，注册 TSF
  6. 成功时不 pause，直接结束；只有失败才暂停

相关脚本

文件作用
tools\dev-stop-pime.ps1 预检停进程；-Auto 跳过交互；DLL 仍加载时 exit 2
tools\dev-uninstall.ps1 注销 TSF、清注册表；DLL 被锁时保留安装目录（KeepInstallRoot），不硬删
tools\dev-install.ps1 复制构建产物到 C:\Program Files (x86)\YIME；DLL 被锁时用 -AllowLocked 跳过替换并警告
dev-*.ps1（根目录） 薄包装，转发到 tools\ 下同名脚本
简化版已废弃：「直接 uninstall → install、DLL 被锁就失败」的旧流程会浪费大量时间。 AGENTS.md 已写明：AI 不得把重装脚本改回简化版。

为什么不用 NSIS 安装包做开发重装？

音元拼音依赖 go-backend。installer\YIME-*-setup.exe 不会打包 go-backend。

日常开发测试只用 Reinstall-PIME-Test.cmd。

验证是否装对

  • 语言栏：中西文 / 设置 / 工具
  • 输入 q → 音元编码（如 巴 ~d），不是 luna_pinyin（去、其）
  • 日志：%LOCALAPPDATA%\PIME\Logs\go_backend.log
  • Rime 配置：%APPDATA%\PIME\Rime\
  • 安装目录：C:\Program Files (x86)\YIME\go-backend\server.exe 时间戳应晚于本次编译
源码改对了但行为没变：先确认 server.exe 已重装，且 PIMELauncher / server 进程已重启。旧进程会让修复看起来无效。

常见问题

DLL 被锁（explorer / Cursor / 浏览器）

强制版会自动原地安装；go-backend 等未锁文件会更新。被锁的 PIMETextService.dll 可能暂时换不了，需重启后再重装才能彻底替换 DLL。

Build.cmd：Application Control (os error 4551)

Windows 智能应用控制拦截了 build\Win32\Release\cargo\ 下的 Rust build-script。

当前 build.bat：Win32 只编 PIMETextService；PIMELauncher 用 PIMELauncher\ 下 cargo build（target\ 目录，通常不受拦）。

不要设置 CARGO_TARGET_DIR 指到 .tmp\，也可能被拦。仍失败则在安全中心关闭智能应用控制或为 C:\dev\Yime 加信任。

ARM64 编译

未检测到 arm64\cl.exe 时会跳过 ARM64（不影响日常 x86/x64 开发）。需要时在 VS Installer 为 C:\BuildTools 添加 MSVC v143 - VS 2022 C++ ARM64 build tools。

CI 与本地 build.bat 不同

GitHub Actions 用 .github\workflows\ci.yaml 直接跑 cmake，不调用根目录 build.bat。本地 build.bat 的 PIMELauncher cargo 绕过、ARM64 检测等改动不影响 CI。

不建议的做法

  • 在 C:\WINDOWS\system32 里跑 .\Reinstall-PIME-Test.cmd（必须先 cd C:\dev\Yime）
  • 只改根目录 Reinstall-PIME-Test.cmd 而不改 tools\templates\（下次 Build 会被覆盖）
  • 把重装脚本改回「只 uninstall + install」简化版
  • 用 installer\*.exe 做开发重装（缺 go-backend）
  • 全量 Build.cmd 时又单独跑 go-backend\build.bat（已包含）
