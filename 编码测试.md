# 编码测试

这份清单用于反复测试当前输入法原型的候选翻页、首选上屏、同音字/词选择，以及 `Home` / `PageUp` / `PageDown` / `End` 翻页行为。

数据来源：

- `yime/reports/runtime_candidates_by_code_true.json`
- `yinjie_code.json`
- `internal_data/yinjie_runtime_key_symbol_mapping.json`
- `internal_data/manual_key_layout.resolved.json`

说明：

- `物理键面` 是按当前键盘布局换算后的实际按键顺序。
- `Shift+...` 和 `AltGr+...` 表示需要对应修饰键。
- 如果当前键盘没有独立 `AltGr` 键，而是用 `Ctrl+Alt` 代替 `AltGr`，则文中的 `AltGr+...` 一律按 `Ctrl+Alt+...` 执行。
- 槽位序列和物理键面以当前运行时编码链路为准；如果测试样例与运行时实际编码不一致，应以运行时编码为准并回写这份清单。
- 词语编码按音节分组显示，组与组之间用 `/` 分隔。
- 当前运行时词语库里，同码词语最多只有 `5` 个，还没有超过 `9` 个的词语组；超过 `9` 的翻页测试请优先用下面的单字组。

## 测试前设置

建议每次手测前先固定同一套环境，避免把焦点问题、窗口问题和编码问题混在一起：

1. 先启动输入法原型，再打开一个简单外部编辑器窗口，优先用记事本或 VS Code 空白编辑区。
2. 确认候选框已进入可见/可唤醒状态，而不是完全退出状态。
3. 如果本轮主要测回贴链路，不要同时改动键盘布局、候选页大小或回贴方式。
4. 如果本轮主要测翻页和拣字，优先使用单字组，不要混入词语组。

## 本轮重点验收项

每轮手测至少记录下面 6 项是否通过：

| 项目 | 通过标准 |
| --- | --- |
| 候选弹出 | 输入完整编码后，候选框能稳定出现，不跑到异常位置 |
| 首选上屏 | `Space` / `Enter` 能一次送出首选，不需要第二次确认 |
| 翻页行为 | `Home` / `PageUp` / `PageDown` / `End` 与鼠标翻页结果一致 |
| 快捷拣字 | `` ` - = \ `` 能稳定选择当前页第 2 到第 5 候选 |
| 回贴链路 | 选中候选后，文本能送达到外部编辑器，不丢字、不重复上屏 |
| 状态恢复 | 送字后候选框、缓冲区和待命状态恢复正确，不影响下一轮输入 |

## 第二版测试单（日用链路版）

这一版不再主要回答“单个功能点是否打通”，而是回答“能不能连续用、连着用、用久一点还稳不稳”。

### 目标

- 验证连续多字输入时，候选框、缓冲区、回贴链路和焦点是否能持续稳定工作。
- 验证翻页、换候选、继续输入之间是否能自然衔接，不出现状态错乱。
- 验证较长时间交互后，候选框位置、选中态、翻页态、输入态是否会累积漂移或失效。

### 第二版重点验收项

| 项目 | 通过标准 |
| --- | --- |
| 连续输入链路 | 连续输入多个字或词时，不需要额外清空或重开候选框 |
| 翻页后继续输入 | 翻页后还能继续补码、选字、送字，不出现候选态错乱 |
| 高亮选中一致性 | 候选高亮、方向键移动、`Space` / `Enter` 送出三者语义一致 |
| 长时间稳定性 | 连续操作一段时间后，不出现明显漂移、卡死、焦点错位或选中态失真 |
| 外部编辑器兼容性 | 在记事本或 VS Code 中持续输入时，不出现明显吞键、重复上屏或焦点丢失 |

### 第二版建议测试场景

#### 场景 A：连续单字输入

目标：验证“选一字，再接着输下一字”是否自然。

建议步骤：

1. 输入 `yi4`，任选一候选送出。
2. 不关闭候选框，立刻继续输入 `li4`、`xi1`、`ji4`。
3. 观察候选框是否保持可继续输入状态，输入缓冲是否正确清空并重新开始。

重点观察：

- 送字后候选框位置是否稳定。
- 高亮候选是否回到合理初始状态。
- 外部编辑器中是否连续正确上屏。

#### 场景 B：翻页后继续输入

目标：验证“翻页浏览”和“继续补码输入”能否共存。

建议步骤：

1. 输入 `yi4` 或 `zhi4`，触发多页候选。
2. 用 `PageDown` / `PageUp` / `Home` / `End` 翻页。
3. 用方向键移动高亮，再用 `Space` / `Enter` 送出。
4. 送出后立刻继续输入下一组编码。

重点观察：

- 翻页后高亮是否总能回到新页首项。
- 方向键移动是否仍然有效。
- 送出后下一轮输入是否从干净状态开始。

#### 场景 C：词语与单字混合连续输入

目标：验证“词语输入”和“单字输入”在同一轮操作里切换是否稳定。

建议步骤：

1. 输入 `zheng4 shi4`，送出词语。
2. 紧接着输入 `yi4` 或 `shi4`，送出单字。
3. 再输入 `man4 tiao2 si1 li3` 或 `he2 shi4`，送出词语。

重点观察：

- 单字 / 词语切换时候选态是否正确刷新。
- 是否出现上一轮候选残留、缓冲区残留或错误回贴。

#### 场景 D：长时间交互稳定性

目标：验证不是“只测几次可以”，而是连续操作后仍然稳定。

建议步骤：

1. 选取 5 到 10 分钟连续测试窗口。
2. 混合使用以下动作：输入、翻页、方向键移动高亮、鼠标点击候选、`Space` / `Enter` 送字。
3. 中途多次切换候选页、多次选择不同页候选，并继续输入下一组编码。

重点观察：

- 候选框是否逐步偏移。
- 高亮是否还跟得上当前页。
- 长时间后是否出现按键无响应、焦点跑偏、候选不刷新、重复上屏。
