# Tests

`tests/` 目录当前按测试职责分层维护：

- 顶层 `test_*.py`：保留跨模块、通用工具层或暂时不适合再细分的小型自动化测试。
- `input_method/`：输入法运行面、候选排序、连续输入、局部优先级和相关运行时辅助测试。
- `yinjie/`：音节编码、双向校验、阶段验证与入口清单测试。
- `syllable_analysis/`：声母/干音编码与分类验证测试。
- `manual/`：需要人工环境、外设或交互配合的手工测试，不纳入默认 pytest 递归运行。

新增测试时，优先放到最贴近其运行面的专题目录；只有在测试职责确实跨目录边界时，才保留在 `tests/` 顶层。
